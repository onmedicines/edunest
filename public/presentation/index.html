Problem 01

Watching a video together
means someone shares their screen.

Screen sharing encodes your entire display and streams it — compressed video of compressed video, for every watcher. It's inefficient by design.

🎮 Discord — Screen Share ~3–5 Mbps upload per viewer
📚 EduNest — Synced Player ~0 Mbps extra (YouTube handles it)

EduNest's approach: Everyone loads the same YouTube video independently. Play, pause, and seek are broadcast as tiny events (~100 bytes) over WebSocket. Each client's YouTube player responds in real time.

Feature 01

Synced Video Player

Anyone can play, pause, or seek — everyone follows in real time.

How it works
1 User pastes a YouTube URL → YouTube IFrame API loads the video locally in each browser
2 Play / pause / seek → onStateChange fires → broadcasts video:state event over Supabase Realtime
3 All peers receive { videoId, isPlaying, currentTime, triggeredBy } → call seekTo() + play/pause
4 ignoreEventsRef suppresses own player events for 500ms after a remote update — prevents feedback loops
// Broadcast play/pause to room const handleVideoChange = (newState) => { setVideoState(prev => ({...prev, ...newState})); channel.send({ type: "broadcast", event: "video:state", payload: { ...newState, triggeredBy: currentUser.id } }); }; // Receive and apply from peers channel.on("broadcast", { event: "video:state" }, ({ payload }) => { if (payload.triggeredBy === currentUser.id) return; setVideoState(payload); } );
⚡ Zero bandwidth overhead. YouTube CDN delivers the video to each client independently.
Feature 02

Real-time Collaborative Notes

Markdown editor. Everyone sees edits live. Download when the session ends.

Edit mode Split mode Preview mode
1 Any keystroke → broadcast(NOTES_UPDATE, { content, updatedBy }) fires immediately
2 Peers check updatedBy !== currentUser.id before applying — prevents overwriting your own typing
3 500ms debounce → saveNotes() Server Action → persists to Supabase DB
4 Download button: client-side Blob → URL.createObjectURL() → .md file

And More

Everything a study session needs.

💬

Room Chat

Persistent messages with emoji reactions (👍🔥💡❤️😂🎉). Typing indicators via Supabase Presence. 2000 char limit.

🎙

Voice — Push to Talk

WebRTC P2P mesh. Hold Space to speak. Supabase Broadcast acts as the WebRTC signaling channel. Google STUN for NAT traversal.

⏱

Shared Timer

Pomodoro-style countdown. Start/pause/reset syncs to all users. State persists across refreshes. Chime via Web Audio API at zero.

✅

Session Goals

Shared todo list with progress bar. Add, check off, or remove goals. Changes broadcast to all users instantly.

🔗

Resource Library

Paste links to YouTube, GitHub, Google Docs, PDFs, Notion. Auto-detected type icons. Inline iframe preview.

✋

Hand Raise + Presence

See who's online in real time. Raise hand to signal a question. Guest login — no account required.
